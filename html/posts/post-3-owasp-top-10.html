OWASP Top 10 Explained — What Every Dev Needs to Know

By Oriana • Mar 20 2026 • 8 min read • #security #certifications

The OWASP Top 10 is a standard awareness document for web application security. It represents the broadest consensus about the most critical security risks to web applications. If you are writing code that runs on the web, this list is not optional reading — it is the baseline.

Published by the Open Web Application Security Project (OWASP), it is updated roughly every three to four years as the threat landscape evolves. The current edition reflects the state of real-world vulnerabilities collected from hundreds of organisations worldwide. Let's walk through the most impactful ones and what you can actually do about them.

A01 — Broken Access Control

The number one risk. Access control enforces policy such that users cannot act outside of their intended permissions. When these controls fail, attackers can access other users' accounts, view sensitive data, or modify data they should not touch. The most common issue is trusting that only authorised users will ever discover or call a given endpoint — they will not. Attackers enumerate APIs, inspect JavaScript bundles, and probe every route they can find.

// Never rely on client-side checks alone
// BAD: The button is hidden in the UI, but the endpoint is unprotected
app.delete('/api/admin/user/:id', (req, res) => {
  // No auth check — any logged-in user can reach this
  db.deleteUser(req.params.id);
});

// GOOD: Always verify server-side
app.delete('/api/admin/user/:id', requireRole('admin'), (req, res) => {
  db.deleteUser(req.params.id);
});

A02 — Cryptographic Failures

Previously called "Sensitive Data Exposure", this covers failures related to cryptography — or the absence of it. Transmitting passwords or tokens over plain HTTP, storing passwords with weak hashes (MD5, SHA-1), or using deprecated algorithms (DES, RC4) all fall here. The fix is straightforward: use TLS everywhere, hash passwords with bcrypt or Argon2, and never roll your own crypto.

A03 — Injection

SQL injection, command injection, LDAP injection — any time user-supplied data is sent to an interpreter without sanitisation, you have an injection vulnerability. Despite being decades old, injection attacks are still devastatingly common because they are easy to introduce and sometimes subtle to spot.

// BAD: user input directly in query
const query = `SELECT * FROM users WHERE name = '${req.body.name}'`;

// GOOD: parameterised query
const query = 'SELECT * FROM users WHERE name = ?';
db.execute(query, [req.body.name]);

A05 — Security Misconfiguration

This is the most commonly seen issue in practice. Default credentials left unchanged, unnecessary features enabled, directory listing exposed, stack traces visible in production, missing security headers — all of these count. A solid security misconfiguration checklist should be part of every deployment pipeline. Start with: disable debug mode in prod, set Content-Security-Policy, X-Frame-Options, and Strict-Transport-Security headers, and remove all default admin accounts.

A07 — Identification and Authentication Failures

Weak passwords, no rate limiting on login endpoints, session tokens that don't expire, credentials in URLs — this category covers all the ways authentication can go wrong. The mitigations are well-established: multi-factor authentication, strong password policies enforced server-side, short session lifetimes, and secure cookie attributes (HttpOnly, Secure, SameSite=Strict).

A10 — Server-Side Request Forgery (SSRF)

SSRF lets an attacker trick the server into making HTTP requests to an arbitrary destination — including internal services behind a firewall, cloud metadata endpoints, or other parts of your infrastructure. It became prominent enough to earn its own Top 10 category after real-world attacks on major cloud providers. The defence: validate and allowlist URLs before fetching, never forward raw user input to internal HTTP clients.

Where to Go From Here

Reading the list is step one. The OWASP project also provides testing guides, cheat sheets, and the OWASP WebGoat deliberately vulnerable application for hands-on practice. If you are working toward a security certification — OSCP, CEH, or CompTIA Security+ — the Top 10 is foundational material. If you are just building websites, it is still the most efficient hour of security education you will spend this year.
